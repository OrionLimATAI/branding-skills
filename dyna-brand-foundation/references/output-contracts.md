# Dyna.Ai Output Contracts

Use these formats to keep brand work useful for designers, marketers, sales teams, vendors, and non-design teammates.

## Visual Direction

Return:

- Objective
- Audience and channel
- Source of truth used
- Message hierarchy
- Visual direction
- Layout or composition notes
- Brand tokens
- Image-generation-ready prompt structure when useful
- Proof level and claim risk
- CTA clarity
- Delivery notes
- Review checklist

## Design Spec

Return:

- Purpose
- Canvas, dimensions, ratio, and viewing context
- Information hierarchy
- Logo placement and clear space
- Typography
- Color tokens and semantic color use
- Imagery, icon, and data-visualization guidance
- Responsive or size adaptation rules
- Export or handoff requirements
- Brand checks

## Vendor Checklist

Return:

- Asset name and use case
- Dimensions and material
- Required files and formats
- Logo clear space and minimum size
- Color, print, lighting, and screen notes
- Installation or production risks
- Proofing steps
- Approval owner and release status

## Page Structure

Return:

- First viewport structure
- Section order
- Hero message and proof module
- CTA flow
- Responsive behavior
- Claim risk
- Visual system guidance
- Development handoff notes

## Review / Audit

Return:

- Overall assessment
- Source of truth used
- Visual quality assessment
- Priority issues using P0/P1/P2/P3
- Suggested fixes
- Production risks
- Approval status
